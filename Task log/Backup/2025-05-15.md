- [x] 김량표 - INIT TRANS 관련 ITL Wait 값 확인 결과
table 에 비해서 index의 1개의 블럭에 저장되는 row 데이터가 많고(일부 컬럼만 저장), index는 정렬되어 있어 insert 시 index 끝의 block에 데이터가 몰리는 경향이 있어서 그런 것 같아요.

그리고, backup이나 획득 같이 동시에 여러 프로세스가 작업하는 index에 wait 가 많이 발생하네요.

이 값들은 db open 하고 난 뒤 누적 값이라서, 값 자체에 의미를 두는 것 보다 top 쪽에 중복되는 것 위주로 식별해서 파라미터를 조정 하는 것이 좋을 것 같아요.

SCH 같은 경우는 사이트의 DBA가 따로 조정 하는 것 같았어요. DB Script에서 생성하는 파라미터 값과는 다르게 설정 되어 있더라고요.
