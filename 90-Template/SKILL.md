# Obsidian 문서 분석 및 최적화 Skill

이 skill은 Obsidian 문서의 내용을 분석하여 다음을 자동으로 추천하고 적용합니다:
- ✅ 적절한 태그 (YAML frontmatter)
- ✅ 문서 제목 (파일명 및 H1 헤딩)

## 태그 가이드 위치

**중요**: 태그 가이드는 항상 다음 경로에 있습니다:
- `30-Resources/Guide/Obsidian Tag Guide.md`

이 파일을 참조하여 기존 태그 체계를 파악하세요.

## 작업 순서

### 1. 문서 읽기
- 사용자가 파일 경로를 제공한 경우: 해당 파일 읽기
- 파일 경로가 없는 경우: 사용자에게 어떤 문서를 분석할지 물어보기

### 2. 태그 가이드 읽기
- `30-Resources/Guide/Obsidian Tag Guide.md` 파일 읽기
- 태그 체계, 계층 구조, 사용 예시 파악
- 각 태그 카테고리의 의미 이해

### 3. 문서 분석
다음 관점에서 문서를 분석하세요:

**필수 분석 항목:**
- **type**: 문서의 종류 (guide, reference, technical-doc, architecture, spec, meeting-note, daily-log 등)
- **내용**: 문서의 주요 주제와 키워드
- **목적**: 문서가 작성된 목적 (교육, 참고, 설계, 문제해결 등)
- **대상**: 누구를 위한 문서인가 (team, developer, admin, beginner 등)
- **프로젝트**: 특정 프로젝트와 관련이 있는가
- **상태**: 문서의 현재 상태 (draft, approved, in-progress 등)

**제목 분석:**
- **현재 파일명**: 파일 경로에서 파일명 추출
- **현재 H1 헤딩**: 문서의 첫 번째 `# ` 헤딩
- **일치 여부**: 파일명과 H1 헤딩이 일치하는지 확인
- **명명 규칙**: 기존 문서들의 파일명 패턴 파악

### 3-1. 제목 명명 규칙 (Obsidian)

**파일명 원칙:**
1. **명확성**: 문서 내용을 명확히 표현
2. **간결성**: 가능한 짧고 간결하게 (권장: 3-8단어)
3. **일관성**: 프로젝트 전체에서 일관된 명명 규칙 사용
4. **특수문자**: 파일 시스템 호환성을 위해 제한적 사용
   - 허용: `-`, `_`, `()`, 공백, 한글, 영문, 숫자
   - 피해야 할 문자: `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`

**일반적인 패턴:**
- 가이드: `[주제] Guide`, `[도구] 사용법`
- 기술 문서: `[기술명] 설계`, `[프로젝트명] 아키텍처`
- 계획: `[주제] 계획`, `[연도] [주제]`
- 회의록: `[날짜] [주제] 회의록`
- 일일 로그: `[이름]`, `[날짜]-[주제]`

**H1 헤딩:**
- 파일명과 동일하거나 더 자세한 버전
- 문서의 핵심 주제를 한 문장으로 표현
- 특수문자 제한 없음

### 4. 제목 추천

문서 내용을 기반으로 적절한 제목을 추천하세요:

**추천 시 고려사항:**
1. 문서의 핵심 내용을 정확히 반영(기존 제목도 고려)
2. 기존 프로젝트의 명명 규칙과 일관성 유지
3. 검색 및 탐색이 용이하도록 명확한 키워드 포함
4. 너무 길지 않게 (권장: 3-8단어)

**추천 형식:**
```markdown
## 📝 제목 추천

### 현재 제목
- **파일명**: `현재파일명.md`
- **H1 헤딩**: # 현재 H1 헤딩
- **문제점**: 파일명과 H1이 불일치 / 내용을 정확히 반영하지 못함 / 등

### 추천 제목
- **파일명**: `새로운-파일명.md`
- **H1 헤딩**: # 새로운 H1 헤딩
- **이유**: 문서의 핵심 내용인 [주제]를 명확히 표현하며, 프로젝트 명명 규칙과 일치
```

### 5. 태그 추천
다음 형식으로 태그를 추천하세요:

```markdown
## 추천 태그

### 필수 태그
- **type/[타입]**: [이유]

### 프로젝트/주제 태그
- **project/[프로젝트명]**: [이유]
- **topic/[주제]**: [이유]

### 대상/카테고리 태그
- **audience/[대상]**: [이유]
- **category/[카테고리]**: [이유]

### 기술/도구 태그 (선택)
- **tech/[기술명]**: [이유]
- **tool/[도구명]**: [이유]

### 상태 태그 (선택)
- **status/[상태]**: [이유]

## 새로운 태그 제안 (태그 가이드에 없는 경우)

- **새태그명**: 설명 및 추가해야 하는 이유
```

### 6. 새 태그 처리
태그 가이드에 없는 태그를 제안할 경우:
1. 왜 이 태그가 필요한지 설명
2. 태그 가이드의 어느 섹션에 추가되어야 하는지 명시
3. 사용자에게 태그 가이드 업데이트 여부 확인

### 7. 제목 및 태그 적용

사용자가 추천에 동의하면 다음 순서로 적용:

**제목 적용 (필요시):**
1. **H1 헤딩 업데이트**: 문서의 첫 번째 `# ` 헤딩을 추천된 제목으로 변경
2. **파일명 변경**: `git mv` 또는 `mv`로 파일명 변경 (사용자 확인 필수)
   - Git 저장소인 경우: `git mv "old-name.md" "new-name.md"` 사용
   - 그 외: `mv "old-name.md" "new-name.md"` 사용

**태그 적용:**
1. YAML frontmatter가 있는 경우: 기존 tags 배열 업데이트
2. YAML frontmatter가 없는 경우: 새로 생성하여 추가
3. 태그는 알파벳 순서로 정렬

**Frontmatter 예시:**
```yaml
---
author: 신화성
date: 2026-02-09
tags:
  - audience/team
  - category/onboarding
  - project/cloud-native
  - type/technical-doc
  - topic/training
  - status/approved
---
```

### 8. 태그 가이드 업데이트 (선택)
새로운 태그를 추가한 경우:
1. `30-Resources/Guide/Obsidian Tag Guide.md` 파일의 해당 섹션 찾기
2. 알파벳 순서에 맞게 새 태그 추가
3. 사용 예시 섹션에 실제 사용 사례 추가 (선택)

## 중요 원칙

1. **태그는 3-7개 사이**: 너무 많으면 관리가 어렵고, 너무 적으면 검색이 어려움
2. **type은 필수**: 모든 문서에 type 태그는 반드시 있어야 함
3. **일관성 유지**: 기존 태그 가이드의 명명 규칙과 계층 구조를 따름
4. **소문자 사용**: 모든 태그는 소문자로 작성
5. **하이픈 연결**: 여러 단어는 하이픈(-)으로 연결 (예: cloud-native, msa-migration)

## 출력 형식

문서 분석 후 항상 다음 순서로 출력하세요:

1. 📄 **문서 요약** (3-5줄)
2. 📝 **제목 추천** (현재 제목 + 추천 제목 + 이유)
3. 🏷️ **추천 태그** (이유와 함께)
4. ✨ **새 태그 제안** (있는 경우)
5. ✅ **적용 계획** (제목 변경 + 태그 적용)

## 예시

```markdown
## 📄 문서 요약
2026년 개발팀의 Windows에서 Linux/Cloud Native 환경으로의 기술 전환을 위한 교육 계획 문서입니다.
개발자와 운영자를 위한 체계적인 학습 커리큘럼과 일정을 제시하고 있습니다.

## 📝 제목 추천

### 현재 제목
- **파일명**: `교육(2026).md`
- **H1 헤딩**: # 2026년 개발팀 기술 전환 교육 계획

### 추천 제목
- **파일명**: `2026 개발팀 교육 계획 (Cloud Native).md` 또는 `교육(2026).md` (현재 제목 유지)
- **H1 헤딩**: # 2026년 개발팀 기술 전환 교육 계획 (현재 제목 유지)
- **이유**:
  - 현재 파일명은 간결하고 연도를 포함하여 시간순 정렬에 유리
  - H1 헤딩은 문서 내용을 정확히 반영하며 명확함
  - 두 제목 모두 적절하므로 현재 상태 유지 권장

## 🏷️ 추천 태그

### 필수 태그
- **type/technical-doc**: 기술 계획 및 사업 계획 문서

### 프로젝트/주제 태그
- **project/cloud-native**: Cloud Native 전환 프로젝트
- **project/msa-migration**: MSA 마이그레이션 프로젝트
- **topic/training**: 교육 관련 문서

### 대상/카테고리 태그
- **audience/team**: 개발팀 전체 대상
- **category/onboarding**: 신규 기술 온보딩 성격

### 상태 태그
- **status/approved**: 2026년 사업 계획으로 승인됨

## ✅ 적용 계획
1. **제목**: 현재 제목이 적절하므로 변경하지 않음
2. **태그**: YAML frontmatter에 위 7개 태그를 알파벳 순으로 추가
```

## 사용 예시

```bash
# 현재 열린 문서 분석
/suggest-tags

# 특정 문서 분석
/suggest-tags path/to/document.md

# 여러 문서 분석
/suggest-tags path/to/doc1.md path/to/doc2.md
```
